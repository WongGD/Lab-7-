## 📋 Вербальна модель предметної області

Система управління продажами ТОВ «SalesPro» автоматизує повний цикл торгових операцій — від реєстрації клієнта до контролю доставки та отримання оплати.

**Основні учасники:**
- **Клієнти** — фізичні та юридичні особи, що розміщують замовлення. Мають кредитний ліміт, один або кілька телефонів та адрес доставки.
- **Менеджери** — співробітники відділу продажів, що обробляють замовлення. Організовані в ієрархію підпорядкування (рекурсивний зв'язок supervisor → manager).
- **Постачальники** — компанії, що постачають товари до каталогу.
- **Склади** — фізичні місця зберігання товарів під управлінням менеджера.

**Основні процеси:**
1. Клієнт розміщує **замовлення** (orders), яке містить рядки товарів (order_lines).
2. Система перевіряє **залишки** товарів на складах (stock).
3. Бухгалтерія виставляє **рахунок-фактуру** (invoices) і реєструє **оплату** (payments).
4. Склад формує **відвантаження** (shipments), логістика здійснює **доставку** (deliveries).

**Обмеження цілісності:**
- Ціна товару > 0; кількість у замовленні ≥ 1
- Статус замовлення: `new | processing | completed | cancelled | returned`
- Email клієнта/менеджера/постачальника — унікальний
- Дата оплати рахунку > дата виставлення рахунку

## 🏗️ ER-модель (5NF)

```
zip_codes ←── addresses ←── clients ──→ client_phones
                                ↓
managers (рекурс.) ──→ orders ──→ order_lines ──→ products ──→ suppliers
     ↓                   ↓                            ↓           (рекурс.)
warehouses ──→ stock ←──┘       invoices ──→ payments
     ↓
shipments ──→ deliveries
```

## 🔑 Бізнес-правила

| # | Правило | Обмеження T-SQL |
|---|---|---|
| 1 | Ціна товару строго > 0 | `CHECK (price > 0)` |
| 2 | Кількість у рядку замовлення ≥ 1 | `CHECK (quantity >= 1)` |
| 3 | Email унікальний (clients, managers, suppliers) | `UNIQUE (email)` |
| 4 | Кредитний ліміт клієнта ≥ 0 | `CHECK (credit_limit >= 0)` |
| 5 | Зарезервована кількість ≤ загальна кількість | `CHECK (reserved_qty <= quantity)` |
| 6 | Статус замовлення — перелічений набір | `CHECK (status IN (...))` |
| 7 | Дата оплати рахунку > дата виставлення | `CHECK (due_date > issue_date)` |
| 8 | Номер відстеження доставки унікальний | `UNIQUE (tracking_no)` |
| 9 | Каскадне видалення рядків при видаленні замовлення | `ON DELETE CASCADE` |
| 10 | ID генерується автоматично для всіх PK | `IDENTITY(1,1)` |
